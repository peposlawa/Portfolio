Typescript Basics

TypeScript is like JavaScript with superpowers! It adds something called static types, which helps you catch errors early, making your code more reliable. One of the big reasons to use TypeScript is that it makes your development smoother with better support in your code editor â€” think autocompletion and instant error checking.

The key differences from JavaScript? You'll start seeing things like type annotations (where you tell TypeScript what kind of data you're working with) and type inference (where it figures that out for you). Plus, you get interfaces, which let you organize your code in a more structured way.

Ready to get started? Let's dive in!

Many Types

In TypeScript, types refer to the annotations and definitions that help the compiler understand what kinds of values a variable, function, or object can hold. Types are the core feature of TypeScript, as they enforce type safety and help catch errors during development rather than at runtime. This makes the code more reliable and maintainable.

TypeScript provides various types, including basic primitives, complex object types, custom types, and more.

Here's a breakdown of different types:

1. Primitive Types

These include string, number, boolean, null, undefined, and symbol. 
                
                    let firstName: string = "Alice";        // string
                    let age: number = 25;                   // number
                    let isActive: boolean = true;           // boolean
                    let value: null = null;                 // null
                    let notDefined: undefined = undefined;  // undefined
                    let uniqueId: symbol = Symbol('id');    // symbol
2. Array Types

Arrays in TypeScript can be of a specific type or a mix of types. 
                
                    let numberArray: number[] = [1, 2, 3];              // Array of numbers
                    let stringArray: Array = ["Alice", "Bob"];  // Alternative syntax
                    let mixedArray: (number | string)[] = [1, "two"];   // Array of numbers and strings
3. Tuple Types

Tuples allow you to define an array with a fixed number of elements, each with its own type. 
                
                    let person: [string, number] = ["Alice", 25];  // Tuple with a string and a number
4. Union Types

Union types allow a variable to be one of multiple types. 
                
                    let id: string | number;
                    id = "123";  // Valid
                    id = 123;    // Valid
5. Enum Types

Enums allow you to define a set of named constants. 
                
                    enum Color {
                      Red,
                      Green,
                      Blue
                    }

                    let color: Color = Color.Green;  // 1 (Enum value)
6. Any and Unknown Types

'any' disables type checking, while 'unknown' enforces type safety through additional checks. 
                
                    let anything: any = "hello";  // No type checking
                    anything = 123;               // Allowed

                    let maybeUnknown: unknown = "world";
                    if (typeof maybeUnknown === "string") {
                      // Allowed because type is checked
                      console.log(maybeUnknown.toUpperCase());
                    }
7. Void Type

The 'void' type is used for functions that don't return a value. 
                
                    function logMessage(message: string): void {
                      console.log(message);
                    }
8. Never Type

The never type represents values that never occur, such as a function that always throws an error. 
                
                    function throwError(message: string): never {
                      throw new Error(message);
                    }
9. Object Types

You can use objects to specify a structure and enforce type checks on the properties. 
                
                    let person: { name: string; age: number; isEmployed: boolean } = {
                      name: "Alice",
                      age: 30,
                      isEmployed: true
                    };
10. Interface Types

Interfaces define the shape of an object and can be used to enforce a consistent structure. 
                
                    interface User {
                      name: string;
                      age: number;
                      isActive: boolean;
                    }

                    let user: User = {
                      name: "Alice",
                      age: 25,
                      isActive: true
                    };
11. Type Alias

Type aliases create custom types to improve readability and reuse. 
                
                    type StringOrNumber = string | number;

                    let idValue: StringOrNumber;
                    idValue = "abc";   // Valid
                    idValue = 123;     // Valid
12. Generics

Generics allow you to write reusable functions or components that work with different types while maintaining type safety. 
                
                    function identity(value: T): T {
                      return value;
                    }

                    let result1 = identity("hello");  // T is replaced with string
                    let result2 = identity(42);       // T is replaced with number
13. Readonly Modifier

The readonly modifier ensures that properties can't be changed after they are initialized. 
                
                    class Car {
                      readonly model: string;

                      constructor(model: string) {
                        this.model = model;
                      }
                    }

                    let myCar = new Car("Toyota");
                    // myCar.model = "Honda"; // Error: Cannot assign to 'model' because it is a read-only property
14. Union and Intersection Types

Union types allow a variable to take on multiple types, while intersection types combine multiple types. 
                
                    // Union type
                    let myId: number | string;
                    myId = 101;   // valid
                    myId = "abc"; // also valid

                    // Intersection type
                    interface Employee {
                      employeeId: number;
                    }

                    interface Person {
                      name: string;
                    }

                    type EmployeePerson = Employee & Person;

                    let employee: EmployeePerson = {
                      employeeId: 123,
                      name: "John"
                    };
15. Type Assertion

Type assertion allows you to override the inferred type and tell TypeScript what the variable should be. 
                
                    let someValue: unknown = "this is a string";
                    let strLength: number = (someValue as string).length;  // Asserting that someValue is a string
