Typescript Basics

If you're a JavaScript developer curious about TypeScript, this guide is for you! TypeScript adds static typing to JavaScript, helping you catch errors early and write more reliable code. In this tutorial, you'll learn how to set up TypeScript, use its key features, and start improving your code today.

Introduction

TypeScript is like JavaScript with superpowers! It adds something called static types, which helps you catch errors early, making your code more reliable. One of the big reasons to use TypeScript is that it makes your development smoother with better support in your code editorâ€”think autocompletion and instant error checking. The key differences from JavaScript? You'll start seeing things like type annotations (where you tell TypeScript what kind of data you're working with) and type inference (where it figures that out for you). Plus, you get interfaces, which let you organize your code in a more structured way.

Ready to get started? Let's dive in!

I'll walk you through examples of various TypeScript types, including primitives, arrays, tuples, unions, enums, interfaces, generics, and more.

1. Primitive Types

These include string, number, boolean, null, undefined, and symbol. 

                
let firstName: string = "Alice";        // string
let age: number = 25;                   // number
let isActive: boolean = true;           // boolean
let value: null = null;                 // null
let notDefined: undefined = undefined;  // undefined
let uniqueId: symbol = Symbol('id');    // symbol
